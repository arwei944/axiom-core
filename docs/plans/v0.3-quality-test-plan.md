## 1. 测试策略总览

### 1.1 测试金字塔

```
        /\
       /  \    手工探索测试
      /____\   
     /      \  自动化集成测试
    /________\ 
   /          \ 单元测试 + 属性测试
  /______________\
```

### 1.2 测试类型映射

| 测试类型 | 工具 | 覆盖范围 | 执行频率 |
|----------|------|---------|---------|
| 单元测试 | `cargo test` | 函数/模块级别 | 每次提交 |
| 集成测试 | `cargo test --test` | crate 间交互 | 每次提交 |
| 属性测试 | `proptest` | 边界条件 | 每次提交 |
| 并发测试 | `loom` | data race | 每次提交 |
| 格式化检查 | `cargo fmt -- --check` | 代码风格 | 每次提交 |
| Clippy | `cargo clippy -D warnings` | 静态分析 | 每次提交 |
| 架构门禁 | `archcheck`、`gatecheck` | 依赖规则 | 每次提交 |
| 安全审计 | `cargo audit` | 漏洞扫描 | 每日 |
| 基准测试 | `cargo bench` | 性能回归 | 每次 PR |
| 覆盖率 | `cargo tarpaulin` | 代码覆盖率 | 每次 PR |

### 1.3 测试环境

- **OS**: Windows 11 / Ubuntu 22.04
- **Rust**: 1.85+
- **SQLite**: 3.40+
- **内存**: 8GB+
- **磁盘**: 10GB+ 可用空间

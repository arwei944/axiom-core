## 1. Phase 1: 确定性基础设施

### 目标

解决 `now_ns()` 硬编码问题，让所有时间相关操作可注入、可控制、可预测。

---

#### Task 1.1: 定义 Clock trait

**描述**: 在 `axiom-core` 中定义时钟抽象 trait，支持系统时钟和模拟时钟。

**输入/前置条件**:
- 无

**输出/交付物**:
- `crates/axiom-core/src/clock.rs` (新建)
- `crates/axiom-core/src/lib.rs` 添加 `pub mod clock`
- 单元测试文件

**依赖关系**: 无

**验收标准**（必须全部满足）:
- [ ] `Clock` trait 定义，包含 `fn now_ns(&self) -> u64`
- [ ] `SystemClock` 结构体实现 `Clock`，调用 `std::time::SystemTime`
- [ ] `MockClock` 结构体实现 `Clock`，内部使用 `parking_lot::Mutex<u64>`
- [ ] `MockClock` 提供 `advance(delta_ns: u64)` 和 `set(timestamp_ns: u64)` 方法
- [ ] `Clock` trait 标记 `Send + Sync`
- [ ] 单元测试覆盖：
  - [ ] `SystemClock::now_ns()` 返回递增时间
  - [ ] `MockClock::now_ns()` 返回设定值
  - [ ] `MockClock::advance()` 正确推进时间
  - [ ] `MockClock::set()` 正确设置时间
- [ ] `cargo check -p axiom-core` 通过
- [ ] `cargo test -p axiom-core -- clock` 通过

**测试策略**:
```rust
#[test]
fn test_mock_clock_advance() { ... }
#[test]
fn test_mock_clock_set() { ... }
#[test]
fn test_system_clock_monotonic() { ... }
```

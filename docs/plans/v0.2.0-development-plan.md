## 二、架构缺口分析

### 缺口1: Lens 原语缺失

**现状**: 文档声称5个核心原语(Cell/Signal/Lens/Axiom/Witness)，但只有 `LensId` 类型，无实际实现

**影响**: 
- 无法从事件日志按需投影状态
- 5原语故事不完整，影响架构可信度

**修复方向**: 实现完整的 Lens 原语，支持从事件日志到状态的按需投影

---

### 缺口2: Witness 链无持久化

**现状**: `axiom-store` 只有内存后端，重启后审计链丢失

**影响**:
- 无法追溯历史状态变更
- 崩溃后无法恢复到之前状态
- 生产环境无法使用

**修复方向**: 添加 SQLite 和文件系统持久化后端

---

### 缺口3: 编译期→运行期约束断层

**现状**: 宏注入的约束在运行时 dispatch 路径上没有统一验证

**影响**:
- 编译期约束可能被绕过（如直接使用 bus.publish）
- 权限检查、版本兼容性等约束只在宏级别生效

**修复方向**: 在总线层统一集成约束验证

---

### 缺口4: 部分 crate 实现较薄

**现状**: identity/prompt/planner/memory 等 crate 只有基础骨架

**影响**:
- 无法在生产环境实际使用
- 文档声称的功能与实际能力不符

**修复方向**: 深化现有 crate，补齐生产级能力

## 四、架构约束 — 硬约束清单

> 所有约束均为硬约束，违反即编译失败或CI拒绝

### 约束1: Crate层依赖规则（编译期强制）

**规则**: crate at level N may only depend on crates at level >= N

**Crate层级**:
| Level | Crate | 允许依赖 |
|-------|-------|---------|
| 0 | axiom-cli | >=0 (所有crate) |
| 1 | axiom-viz | >=1 |
| 2 | axiom-agent | >=2 |
| 3 | axiom-oversight | >=3 |
| 4 | axiom-runtime | >=4 |
| 5 | axiom-store | >=5 |
| 6 | axiom-macros | >=6 (仅core) |
| 7 | axiom-core | 仅第三方依赖 |

**强制机制**: [crates/axiom-core/src/gate.rs](file:///d:/work/trae/axiom-core-project/crates/axiom-core/src/gate.rs#L58-L77) — `verify_dependencies()` 编译期检查

## 五、约束对开发的硬性要求

### 新增crate时的约束

**步骤**:
1. 在 `CRATE_LAYERS` 中注册新crate和层级
2. 确保依赖方向正确（只能依赖同层或下层）
3. 使用已审核的第三方依赖
4. 不使用禁止的依赖（如 `async-trait`）

---

### 新增Cell时的约束

**步骤**:
1. 使用 `#[cell(layer = "...")]` 宏标注层归属
2. 通过编译期检查（`CanSendTo` trait bound）
3. 运行时自动遵守层间调用规则

---

### 新增Signal时的约束

**步骤**:
1. 使用 `#[signal(kind = "...", layer = "...")]` 宏标注
2. 实现 `Signal` trait
3. 实现 `Schema` trait（验证逻辑）

---

### 新增工具时的约束

**步骤**:
1. 实现 `Tool` trait
2. 通过 `ToolRegistry` 注册
3. 添加权限控制（`required_permission`）
4. 每次调用产生Witness记录
5. 使用 `#[capability(dim = "tool")]` 标注版本

---

### 新增Lens时的约束

**步骤**:
1. 使用 `#[lens]` 宏标注
2. 实现 `Lens` trait
3. 自动注册到 `LENS_REGISTRY`
4. 使用 `#[capability(dim = "schema")]` 标注版本（投影逻辑版本）

## 七、约束违规后果

| 违规类型 | 后果 | 修复方式 |
|----------|------|---------|
| 反向依赖 | 编译失败 | 调整crate层级或依赖关系 |
| 禁止依赖 | 编译失败 | 移除禁止依赖，使用替代方案 |
| 未审核依赖 | 编译失败 | 将依赖添加到 `AUDITED_DEPS` 并审核 |
| 跨层调用 | 编译失败 | 使用合法的调用路径 |
| `unwrap()` / `expect()` | Clippy警告（CI失败） | 使用 `Result` 模式 |
| 缺少文档 | Clippy警告（CI失败） | 添加文档注释 |
| 未注册能力版本 | 兼容性检查失败 | 添加 `#[capability]` 标注 |

# 状态转换图文档

> 本文档描述 Axiom Core 中关键状态机的转换规则，包括 Cell 监督状态、Circuit Breaker、Witness 转换结果等。

---

## 1. Cell 监督状态机 (Supervisor::CellState)

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Cell 监督状态转换                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌──────────┐                                                           │
│  │ Running  │◄─────────────────────────────────────────────────────┐   │
│  └────┬─────┘                                                      │   │
│       │                                                             │   │
│       │  panic / error                                              │   │
│       ▼                                                             │   │
│  ┌──────────────┐    SupervisionStrategy::Restart    ┌─────────────┐ │   │
│  │ Restarting   │───────────────────────────────────►│ Restarting  │ │   │
│  │ {attempt: N} │                                   │ {attempt:   │ │   │
│  └──────┬───────┘                                   │  N + 1}     │ │   │
│         │                                           └──────┬──────┘ │   │
│         │                                                  │         │   │
│         │  success                                         │         │   │
│         └──────────────────────────────────────────────────┘         │   │
│         │                                                             │   │
│         │  max_retries exceeded / Stop / Escalate                     │   │
│         ▼                                                             │   │
│  ┌──────────┐                                                         │   │
│  │ Stopped  │                                                         │   │
│  └──────────┘                                                         │   │
│                                                                         │
│  CircuitBreaker 插入点：                                                  │
│  Running ──[failure_threshold 次失败]──► CircuitOpen ──[reset_after]──► Running │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 状态说明

| 状态 | 说明 |
|------|------|
| `Running` | Cell 正常运行中 |
| `Restarting { attempt }` | Cell 崩溃后正在重启，`attempt` 为当前尝试次数 |
| `CircuitOpen { until }` | 熔断器打开，拒绝新请求直到 `until` 时间点 |
| `Stopped` | Cell 已停止，不再尝试重启 |

### 转换规则

1. **Running → Restarting**: Cell 发生 panic 或返回错误
2. **Restarting → Running**: 重启成功
3. **Restarting → Restarting**: 重启失败，attempt + 1（未超过 max_retries）
4. **Restarting → Stopped**: 重启失败，超过 max_retries，或策略为 Stop/Escalate
5. **Running → CircuitOpen**: 连续失败次数达到 `failure_threshold`
6. **CircuitOpen → Running**: 经过 `reset_after` 时间后自动恢复

## 2. Circuit Breaker 状态机 (CBState)

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Circuit Breaker 状态转换                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   成功请求 / 记录成功                                                    │
│  ┌──────────┐                                                           │
│  │ Closed   │◄──────────────────────────────────────────────────┐     │
│  └────┬─────┘                                                   │     │
│       │                                                          │     │
│       │ 失败次数 >= threshold                                    │     │
│       ▼                                                          │     │
│  ┌──────────┐     reset_after 时间到期    ┌──────────┐           │     │
│  │ Open     │────────────────────────────►│ HalfOpen │           │     │
│  └────┬─────┘                           └────┬─────┘           │     │
│       │                                       │                  │     │
│       │ 再次失败                              │ 成功              │     │
│       └───────────────────────────────────────┘                  │     │
│       │                                                          │     │
│       │ 失败                                                     │     │
│       └──────────────────────────────────────────────────────────┘     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 状态说明

| 状态 | 说明 |
|------|------|
| `Closed` | 熔断器关闭，允许请求通过 |
| `Open` | 熔断器打开，拒绝请求 |
| `HalfOpen` | 半开状态，允许部分请求通过以测试恢复 |

## 3. Witness 转换结果 (TransitionOutcome)

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Witness 转换结果                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   Cell.handle() 执行完成                                                 │
│          │                                                               │
│          ▼                                                               │
│   ┌─────────────┐   成功处理    ┌────────────┐                           │
│   │ Success     │◄─────────────│  正常返回   │                           │
│   └─────────────┘              └────────────┘                           │
│                                                                         │
│          │                                                               │
│          │ 处理失败                                                       │
│          ▼                                                               │
│   ┌─────────────────┐                                                    │
│   │ Failed { reason }│  ← 包含错误原因                                    │
│   └─────────────────┘                                                    │
│                                                                         │
│          │                                                               │
│          │ Axiom 约束违反                                                  │
│          ▼                                                               │
│   ┌─────────────────────────┐                                            │
│   │ AxiomViolated {          │                                            │
│   │   axiom_name,            │  ← 违反的 Axiom 名称                        │
│   │   message                │  ← 具体违反信息                             │
│   │ }                        │                                            │
│   └─────────────────────────┘                                            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 转换说明

| 结果 | 触发条件 | 后续动作 |
|------|----------|----------|
| `Success` | 处理成功完成 | 记录 Witness，发送输出消息 |
| `Failed { reason }` | 处理过程中发生错误 | 记录 Witness，触发 Supervisor |
| `AxiomViolated { ... }` | 违反 Axiom 约束 | 记录 Witness，触发熵治理 |
